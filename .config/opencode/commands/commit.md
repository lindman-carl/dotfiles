# Commit Changes

Commit all uncommitted changes to the current branch using Commitizen conventional commit format.

## Step 1: Analyze Changes

First, understand what has changed:

```bash
git status
git diff --staged
git diff
```

## Step 2: Stage All Changes

```bash
git add -A
```

## Step 3: Create Commit Message

Use **Commitizen conventional commit format**:

```text
<type>(<scope>): <subject>
<body>
<footer>
```

### Types (required)

- `feat`: A new feature
- `fix`: A bug fix
- `docs`: Documentation only changes
- `style`: Code style changes (formatting, semicolons, etc.)
- `refactor`: Code refactoring (no feature or bug fix)
- `perf`: Performance improvements
- `test`: Adding or updating tests
- `build`: Build system or dependencies
- `ci`: CI configuration changes
- `chore`: Other changes (maintenance, tooling)
- `revert`: Reverting a previous commit

### Scope (optional)

- The area of the codebase affected (e.g., `auth`, `api`, `ui`, `db`)

### Subject (required)

- Imperative mood, lowercase, no period at end
- Max 50 characters
- Example: "add user authentication endpoint"

### Body (optional)

- Explain the "what" and "why", not the "how"
- Wrap at 72 characters

### Footer (optional)

- Reference issues: `Closes #123`, `Fixes #456`
- Breaking changes: `BREAKING CHANGE: description`

## Step 4: Commit

Create the commit with a properly formatted message.

**IMPORTANT**:

- Do NOT add any AI/Claude attribution, co-author tags, or "Generated by" messages
- Keep the commit message clean and professional
- The commit should look like it was written by a human developer

## Examples

```text
feat(auth): add JWT token refresh endpoint
Implement automatic token refresh to improve user session management.
Tokens are now refreshed 5 minutes before expiration.
Closes #234
```

```text
fix(api): handle null response from payment gateway
The payment gateway occasionally returns null instead of an error object.
Added null check to prevent crash.
```

```text
chore: update dependencies to latest versions
```

```text
refactor(db): simplify query builder interface
```
